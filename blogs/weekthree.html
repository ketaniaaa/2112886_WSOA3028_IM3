Week 3: What is Semantic Markup and Why is it Important?

ketania govender 3 May 2021

Semantic Markup alludes to creating more meaningful markup when creating websites. Semantic markup is a large feature of HTML5. Semantic markup allows for better structured HTML as it emphasizes the importance of the content over the appearance. Semantic markup makes use of appropriate markup language to create readable and clear webpages that are more accessible to browsers, screen readers, users and developers themselves.

Semantics refers to how the meaning of text is interpreted. Semantic markup is particularly interpreted by browsers and screen readers, therefore are present to provide the browser with clearer information regarding the web application. In order to correctly make use of semantic markup, words must be used in the appropriate context.

This means that with the introduction of HTML5, tags that have semantic meaning have been introduced. This helps web developers move away from depending on ambiguous tags such as spans and divs. These tags do not provide the browser with information regarding the contents of these tags. This therefore makes it difficult to interpret large pieces of html and convolutes the markup. Semantic tags that have been introduced allow for clearer indication regarding the contents of tags.

There can be seen to be two distinct functions of semantic markup in HTML. These include:

  1. Elements must be used in the appropriate purpose as indicated by their semantic tags.
  2. The markup used to style a webpage must be inherently separate from the content of the page.
This means that when creating semantic markup, one must adhere to the intended uses of the semantic tags. An H1 tag should not be used to create a bolded text but should be used to indicate the main heading of a page instead. This means one should choose meaning over appearance and should focus of using CSS instead of HTML elements to create specialized appearance within a web page.

Semantic markup is a crucial aspect of creating web pages. The meaning that semantic markup provides the HTML we create is helpful to a large range of people. As more of the population becomes internet users, there is a larger need for accessible internet practices. This forces the need to create markup that can be available to users that are dependant on screen readers and cannot view the web from a visual point-of-view. Not only is semantic markup useful to internet users, but it makes creating HTML much easier for designers and developers. When the mark-up is populated with undifferentiated divs and spans, it makes it harder to style and edit a webpage. Information is not well-organized and can cause inefficiency. This has been a personal experience of creating web pages. The use of just divs makes the process a lot harder for me as I am confused as to where the relevant information I need to edit or create on an HTML document is, but this also makes styling harder.
